模型吃掉开发框架：Google DeepMind的Agentic时代来了

Training Data

Google DeepMind's Logan Kilpatrick: Why the Model Eats the Harness

Logan Kilpatrick，Google AI Studio和Gemini API的负责人，在播客中展示了一个令人印象深刻的例子：有人用Gemini的Omni功能实时编辑一张照片，让一只狗出现在舞台上，而画面中其他角色的视线和表情都精准配合，甚至他还自然地抚摸狗并继续讲话。他认为这种细腻程度体现了模型对世界理解的进步，而不仅仅是图像生成。这个例子引出了一个更大判断：模型能力正从“回答问题”转向“在复杂场景中执行具体任务”，而这对内容创作和产品设计的影响将是根本性的。

Google内部正在发生一个结构性变化：过去Gemini语言模型是贯穿所有产品的单一主线，现在这个角色正在被Antigravity agent harness取代。Sundar Pichai在IO大会上宣布进入Agentic Gemini时代，核心就是这个agent框架——它不再只是API，而是一个能主动代表用户采取行动的运行环境。所有Google产品——搜索、Gemini App、Cloud、AI Studio——都在重新基于这个harness重构，使其变成“agent原生”。这意味着模型不再是终点，而是被嵌入一个能执行、能迭代、能调用工具的系统中。

Antigravity本身不是一个IDE，而是由核心IDE、Web端agent优先体验、CLI、SDK等多个组件构成的生态系统。它的设计目标是“在任何地方接住开发者”——你可以通过Gemini API使用托管agent，也可以直接使用Antigravity的完整能力。更重要的是，这个harness对不同场景做了差异化：用于AI Studio的编码场景和用于Gemini App的消费场景只有80%的公共部分，其余部分针对具体需求做优化。这些优化决定了agent在真实使用中的效率和可靠性，也是Google区分于其他平台的关键。

关于业务蚕食，Logan的观点很直接：AI能回答问题最初被认为会减少搜索量，但实际上搜索总量反而增加了，代理也在执行搜索，人类也在搜索，整体生态价值是正向的。短期一两年内人类行为模式相对清晰，但三到五年后当技术和产品形态变化更大时，结果难以预测。他认为，这恰恰是产品成功的代价——如果你真的让AI替用户完成更多事，用户花在产品上的直接时间减少是大概率事件，但关键是能否在别处创造更大的价值。
