Google的Agent时代：模型如何吞掉框架

Training Data

Google DeepMind's Logan Kilpatrick: Why the Model Eats the Harness

Google DeepMind的Logan Kilpatrick在访谈中揭示了公司AI战略的转折点。过去，Gemini API是连接Google所有产品的单一线索，现在这个角色被Agent Harness取代。这个代理框架正在成为Google内部所有产品重新构建的基础——从搜索到Gemini应用，再到Cloud和AI Studio，都开始基于同一个代理框架来运行。这意味着Google不再只是提供语言模型能力，而是提供一套能让模型自主执行任务的系统。对开发者来说，这意味着你接入的不再是单纯的API，而是一个具备行动能力的基础设施。

Agent Harness的具体形态是多元的。它既是一个核心IDE，也是一个Web端的代理优先体验，同时提供命令行工具和SDK。Logan强调，Google的策略是“在开发者所在的地方与他们相遇”——你可以通过Gemini API使用托管代理，也可以直接用Agent Harness构建自己的应用。更关键的是，这个框架并非一刀切：用于AI Studio的版本针对“vibe coding”场景做了优化，而用在Gemini应用中的版本则更贴近消费级、全天候在线的代理需求。大约80%的核心功能是共通的，剩下的20%根据场景做专门调优。

关于代理能力是否会导致Google自身业务被蚕食的问题，Logan给出了一个反直觉的判断。最初业界担心AI直接回答问题会减少搜索量，但实际情况是搜索量反而增加了——因为人们在AI帮助下能做更多事，甚至代理本身也在“搜索”。当前阶段，代理和人类行为是正和游戏，整个生态的价值在扩大。但他也坦承，一到两年内的趋势相对清晰，三到五年后当技术进一步成熟、产品形态改变时，人类行为模式会如何演变，现在很难预测。而这恰恰是产品成功的关键——让用户自己发现新的使用方式。
