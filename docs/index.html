Google DeepMind's Logan Kilpatrick: Why the Model Eats the Harness

Training Data

摘要：模型正在吃掉“缰绳”——Google DeepMind的Agent战略转向

Google I/O 上 Sundar 宣布的“Agentic Gemini 时代”并非空话。Logan Kilpatrick 明确指出，过去 Google 旗下近 50 个产品没有统一的 AI 贯穿线，Gemini 模型 API 是第一根线，而现在这条线正在被 Agent Harness（代理缰绳） 替换。对开发者而言，这意味着你不再只是调用一个模型来回答问题，而是调用一个能替你执行任务的代理系统。这个缰绳本身就是一套完整的“行动框架”，它正在被嵌入搜索、Gemini App、Cloud 和 AI Studio 等所有主要产品中——模型本身反而成了这个缰绳的一个组件。换句话说，未来的 Google 产品里，模型是“大脑”，但“手脚”被 Agent Harness 统一接管了。

Antigravity 就是这个缰绳最核心的体现，但千万别只把它当成一个 IDE。Logan 把它描述为一个生态：包含核心 IDE、面向 Web 的 Agent 优先体验、CLI 和 SDK，它试图“在开发者所在的地方接住他们”。更关键的是，Antigravity 内部的同一套 Agent Harness 被不同产品做了行业定制——AI Studio 里的 vibe coding 版本优化了开发场景，Gemini App 里的版本优化了 24/7 的消费者场景。对使用 Google 生态的团队来说，这意味着你可以在不同产品间获得一致的 Agent 能力，但每个产品会针对你的实际工作流（写代码 vs 处理邮件）做差异化优化，而不是一个万能工具打天下。

关于“AI 是否会蚕食 Google 自身业务”的担忧，Logan 的判断很直接：目前来看，AI 让搜索和产品使用量双双增长，这是一个正和游戏。人类搜索更多了，同时 Agent 也在代替人类搜索。短期一两年内，用户行为的变化是清晰的；但三五年后，当技术质变、产品形态大变时，很难预测。对企业决策者的启示是：不要被“AI 取代用户”的恐惧绑架——现阶段更可能发生的是 AI 创造新的人类交互场景，而你需要提前设计好“Agent 替你干活”和“人类亲自干活”之间的切换路径，而不是盲目追求全自动化。
