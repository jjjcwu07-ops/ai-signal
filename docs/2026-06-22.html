Google DeepMind's Logan Kilpatrick: Why the Model Eats the Harness

Training Data

一句话结论：
模型能力正在迅速商品化，真正决定产品胜负的是“智能体框架”——谁能把模型包装成能自主执行任务的系统，谁就掌握了下一波增长。

背景：
Logan Kilpatrick 是 Google AI Studio 和 Gemini API 的负责人，相当于谷歌大模型产品化的“总包工头”。他每天面对的是：如何让几十个谷歌产品（搜索、Gmail、Cloud）都跑在同一个 AI 底座上。他的判断直接决定开发者用什么工具、用户怎么用 AI。

核心洞察（反直觉 + 具体）：

  1. “模型吃掉框架”是错的，实际上是“框架吃掉模型”。

Logan 说得很直白：“代码编写已经被证明是通用的智能体框架。” 意思是，别只盯着下一个更大参数的模型——真正拉开差距的是怎么让模型“动手干活”。现在谷歌把所有产品的 AI 层都统一到了同一个“智能体框架”上（内部叫 Agent Harness），这个框架才是真正的护城河。你之前用的 Gemini API 只是入口，现在 Agent Harness 才是通过线。
对你有什么用？ 如果你是开发者，别再纠结选 GPT-4 还是 Gemini 2.0。先想清楚你要的 Agent 框架是什么——能跑代码、能操作网页、能调用工具。选框架比选模型更重要。

  1. AI 不会减少你的使用时间，反而会创造新需求。

Logan 举了个反直觉的例子：大家曾担心 AI 直接给出答案会杀死搜索，结果搜索量反而增加了。因为 AI 完成任务后，人会诞生更多新任务，而且智能体本身也在替人搜索。
对你有什么用？ 别怕 AI 取代你——那些担心“人类不再浏览网页”的言论是错的。真实情况是：AI 帮你干了杂活，你才有精力去做更高价值的事，同时你还多了一个“数字员工”替你持续执行任务。

  1. 谷歌内部正在用同一个 Agent 框架串联全部产品。

Logan 透露：过去谷歌有 50+ 产品，没有统一的 AI 通过线；现在 Agent Harness 成了这条线，从搜索到 Gemini App 到 AI Studio，都用同一套“自动化大脑”。
对你有什么用？ 这意味着你未来用谷歌产品时，背后的一致性会大幅提升。今天你在搜索里用到的 Agent 能力，明天就会出现在 Gmail 或 Calendar 里。你只需要学一次怎么指挥它。

最难忘的一句话：
“Coding has proved to be the general-purpose agent harness.”（代码编写已被证明是通用的智能体框架。）
—— Logan Kilpatrick，直指本质：真正通用的智能体不是聊天，而是能写程序、执行逻辑的系统。
